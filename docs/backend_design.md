# バックエンド詳細設計書 (Kondate Navi Pipeline)

このドキュメントでは、浦安市学校給食の献立PDFからデータを抽出し、Webアプリへと配信するバックエンド処理（Pythonパイプライン）の内部実装詳細について解説します。

---

## 1. 全体アーキテクチャと処理フロー (`src/run_pipeline.py`)

本システムのマスターオーケストレーターです。他のPythonファイル群を `subprocess.run` で順番に独立したプロセスとして呼び出します。途中のプロセスが異常終了（0以外のExit Code）を返した場合、パイプライン全体が直ちに安全停止します。

- **引数なしの場合**: 実行時のシステム時刻 (`datetime.now()`) を基準に、自動的に「翌月」のデータをターゲットとします。
- **引数ありの場合**: 
    - `--year YYYY`: ターゲット年を指定。
    - `--month M`: ターゲット月を指定。
    - `--push`: 処理成功後、自動的に GitHub へ `git push` を実行します（データの公開）。

### 呼び出しシーケンス
1. **[Step 1] PDFのダウンロード** (`src/download_menu.py`)
   市のサイトから第一・第二（小学校）および第三（中学校）のPDFを自動取得。
2. **[Step 2] AI文字起こしと画像解析** (`src/transcribe_pdf.py`)
   Gemini 3.1 Pro でメニュー・食材をパース。OpenCVでお箸マーク（〇）を座標検知。
3. **[Step 2.5] データのエンリッチメント** (`src/enrich_menu.py`)
   AIで献立に応じた絵文字を付与し、主菜（★）フラグを立てる工程。
4. **[Step 3] データの集計・結合** (`src/generate_data.py`)
   複数の調理場データを1つの月次JSONに集約。
5. **[Step 4] データの公開** (`src/publish_data.py`)
   GitHub Pages のデータ用ディレクトリへ配置・Push。

---

## 2. モジュール詳細解説

### 2.1 PDF自動取得スクリプト (`src/download_menu.py`)
- **呼び出し引数**: `python src/download_menu.py [year] [month]` (引数なしの場合は当月)
- **目的**: 浦安市の公式ページをスクレイピングし、対象年月の「小学校（第一・第二）」および「中学校（全校共通・第三調理場）」の献立PDFを網羅的にダウンロードします。
- **実装詳細 (2段階スクレイピング)**:
  1. **インデックスページの探索**: まず固定の「学校給食」インデックスページ (`index.html`) をスクレイピングし、「給食献立」というテキストが含まれるリンクを動的に探して詳細ページへ遷移します。
  2. **PDFの抽出**: 遷移先の詳細ページで和暦変換された見出しブロックを探し、「第一調理場」「第ニ調理場」「第三調理場」のすべてのPDFリンクを取得します。
  - 取得したPDFは `downloads/YYYYMM/` ディレクトリに保存されます。
  - **エラー検知**: 対象月のリンクが1つも見つからなかった場合（市役所がまだ公開していない場合）、直ちに `sys.exit(1)` を呼び出して異常終了します。これにより上位のパイプラインも空打ちを検知して安全に停止します。

### 2.2 画像解析・AI文字起こし (`src/transcribe_pdf.py`)
- **目的**: PDFを画像化し、Gemini（文字起こし）とOpenCV（お箸マーク検知）の2系統でデータを抽出します。
- **全体制御 (フォルダ一括ループ)**:
  - `--year` と `--month` 引数で指定された月（例: `downloads/202603/`）のフォルダ内に存在する **すべてのPDFファイルを自動的に検知し、順次ループ処理** で解析を実行します。これにより「第一調理場」「第二調理場（および将来機能拡充時の第三調理場）」といった複数のPDFがあっても、引数1つの指定だけで全てのファイルが網羅的に処理される設計になっています。
- **AI文字起こし (Gemini 3.1 Pro)**:
  - PDFの1ページ目を `PyMuPDF` で300DPI相当のPNG画像としてレンダリングします（透過背景対策済み）。
  - 画像と「プロンプト」を `Vertex AI` の Gemini エンドポイントに送信します。プロンプトには厳格なJSONスキーマと、「ひらがなは漢字に直す」「表の外の情報は無視する」「献立名は勝手に正規化せず原型をとどめる」といった強力な抽出制約が書かれています。
- **お箸フラグの画像解析判定 (OpenCV)**:
  AIによる「手書き丸印の認識」は不確実なため、ピクセルベースの幾何学的な画像処理で `needs_chopsticks` を判定します。
  1. **動的アンカーの発見**: まずファイル内のテキスト（Words）から「日」というヘッダーのX座標を探し出します。
  2. **真円の抽出**: `cv2.findContours` で輪郭を抽出し、真円度（面積と半径の比率）が `0.5 < x < 1.2` であり、かつ大きさが適切（半径20〜50px）な「丸印（〇）」だけをリストアップします。この時、「日」列のX座標周辺にある〇だけを対象にします。
  3. **日付とのマッチング**: カレンダー上の数字（1〜31）の座標と、発見された丸印の中心座標の直線距離を計算します。距離が60px以内であれば重なっていると判定し、その日付の `needs_chopsticks` を `True` とします。

### 2.3 献立エンリッチメント (`src/enrich_menu.py`)
- **目的**: AI（Gemini）を活用し、パース済みの献立テキストに「絵文字」を付与し、「主菜（★）」を特定します。
- **処理ロジック**:
  - `transcribe_pdf.py` で生成された各日・各施設のメニュー名を読み込みます。
  - **絵文字付与**: 独自のキーワード優先順位（例：肉料理より魚料理を優先する等）に基づき、料理内容に最も適した絵文字を自動的にマッピングします。
  - **主菜判定**: メニュー名の中からその日の「メインディッシュ」を判別し、名前に `★` マークを付与します。これにより、フロントエンド側でアクセント表示（オレンジのバー）を行うためのフラグが確立されます。

### 2.4 データ集計・ファイル整形 (`src/generate_data.py`)
- **呼び出し引数**: `--year YYYY --month M` 
- **目的**: `transcribe_pdf.py` が吐き出した複数の施設別JSONデータを、1つの構造化ファイル (`data/YYYY_M.json`) に結合整理します。
- **処理対象**: 指定された年月の `results/YYYYMM/*.json` だけを読みに行きます。
- **実装詳細**:
  - 読み込んだJSONのメタ情報 `meta.facility_name` (第一調理場・第二調理場など) を、日次データ（`daily_menus`）の配列内に直接コピー注入します。
  - 同様に、Webアプリ内でのフロントエンド処理（ReactのKeyやソート）を容易にするためのID (`date_id`: "2026-3-2") や `year`, `month` といった一意キーを追加します。
  - これにより、施設ごとにネストされていたデータ構造が、「日付と施設ごとに完全に独立したフラットな配列構造」へと最適化されます。

### 2.5 GitHub Pages への公開 (`src/publish_data.py`)
- **目的**: 生成された JSON を GitHub の公開リポジトリへ反映させ、Web アプリからアクセス可能にします。
- **処理プロセス**:
  1. **ローカル環境へのコピー**: 集計済みの JSON を `webapp/public/data/` ディレクトリへコピーします。これにより、ローカル開発環境でも最新データが即座に利用可能になります。
  2. **データ整合性バリデーション**: 書き出された JSON が空でないか、正しくパースできるかを最終チェックします。
  3. **Git 同期 (Push)**: Git コマンドを実行し、`data` および `webapp/public/data` ディレクトリの変更を `main` ブランチへコミット＆プッシュします。
- **公開メカニズム**: 
  GitHub リポジトリへプッシュされた JSON フォルダは、`https://droprop.github.io/kondate-navi/` 経由で静的ファイルとして公開され、Web アプリからリアルタイムにフェッチされます。
