# システム詳細設計書 (Kondate Navi Pipeline)

このドキュメントでは、浦安市学校給食の献立PDFからデータを抽出し、Webアプリへと配信するバックエンド処理（Pythonパイプライン）の内部実装詳細について解説します。
開発者がソースコードの全体像や各スクリプトの役割・内部ロジックをリバースエンジニアリング的に把握するための設計書です。

---

## 1. 全体アーキテクチャ (`src/run_pipeline.py`)

全体を統括するマスターオーケストレーターです。指定された（または自動計算された来月の）年月を受け取り、以下の4つのステップを直列で実行します。途中でエラーが発生した場合（Exit code 1など）、後続処理は実行されずパイプライン全体として異常終了します。

1. **Step 1**: `download_menu.py` (PDF取得)
2. **Step 2**: `transcribe_pdf.py` (AI解析・画像処理)
3. **Step 3**: `generate_data.py` (JSON結合・整形)
4. **Step 4**: `upload_to_firebase.py` (デプロイ)

---

## 2. モジュール詳細解説

### 2.1 PDF自動取得スクリプト (`src/download_menu.py`)
- **目的**: 浦安市の公式ページをスクレイピングし、対象年月の「小学校給食（第一調理場・第二調理場）」の献立PDFをダウンロードします。
- **実装詳細**:
  - `BeautifulSoup` を使用して指定URLをパースします。
  - 「令和X年Y月分献立予定表」という和暦変換された見出しブロックを探し、その直下にある「第一調理場」「第ニ調理場」のPDFリンク (`<a>` タグ) を抽出します。
  - 取得したPDFは `downloads/YYYYMM/` ディレクトリに保存されます。
  - **エラーハンドリング**: 対象月のリンクが1つも見つからなかった場合はその場で `exit(1)` となり、処理を中断します。

### 2.2 画像解析・AI文字起こし (`src/transcribe_pdf.py`)
- **目的**: PDFを画像に変換し、AI（Gemini）と画像解析（OpenCV）の2つのアプローチで情報を抽出します。**本システムのコアエンジンです。**
- **実装詳細**:
  1. **PDFの画像化**: 
     `PyMuPDF (fitz)` を使用してPDFの1ページ目を300DPIでPNG形式にレンダリングします (`images/YYYYMM/` に一時保存)。
  2. **AI文字起こし (Gemini)**: 
     画像を `Gemini 3.1 Pro (Vertex AI)` に送信します。強力なプロンプトエンジニアリングにより、表形式の画像から「日付・メニュー・三色食品群・栄養価」を抽出し、正確なJSON配列にパースさせます。
  3. **お箸フラグの判定 (OpenCV)**:
     AIによる「図形の認識（カレンダーに書かれた手書き風の丸印）」は精度が不安定なため、確実性を重視して古典的な画像処理（OpenCV）を併用しています。
     - `cv2` で画像をグレースケール・2値化し、コントラストを調整します。
     - 「箸」という文字のピクセル座標を正規表現や既存情報から特定し、その周辺のカレンダー枠を計算します。
     - 各日付のセル内に一定以上の黒ピクセル面積（丸印）が存在するかを判定し、`needs_chopsticks: true/false` を付与します。
  - 最終的にAIのテキスト情報とOpenCVの論理フラグを合体させ、PDFごとに `results/YYYYMM/` にJSONファイルとして保存します。

### 2.3 データ集計・ファイル整形 (`src/generate_data.py`)
- **目的**: 複数に分かれた調理場ごとのJSONデータを、Webアプリが直接読み込みやすい1つのファイル (`data/YYYY_M.json`) に結合・整形します。
- **実装詳細**:
  - `results/YYYYMM/` 内の全JSONを読み込みます。
  - AIが生成した親階層の `meta.facility_name` (施設名) を、1日ごと（`entry`ごと）の配列データの中に直接埋め込みます。
  - UI側での検索・フィルタリング・マッピングをO(1)や単純ループで行えるように、ユニークな `date_id` (例: "2026-3-2") や `year`, `month` キーを新規付与し、完全にフラットな配列構造に作り変えます。

### 2.4 Firebaseへのデプロイ (`src/upload_to_firebase.py`)
- **目的**: 集計済みの月別ローカルJSONデータを、本番の Firebase Cloud Storage に公開します。
- **実装詳細**:
  - `firebase-admin` SDKを使用し、アプリケーションのデフォルト認証情報(ADC)を通じてGoogle Cloudに接続します。
  - 引数で渡された特定の月ファイル（例：`data/2026_3.json`）単体を Storage 上の同じパスへ上書きアップロードします。
  - **キャッシュ対策の要**: 
    アップロード時にHTTPヘッダーとして `Cache-Control: no-store, max-age=0` メタデータを付与しています。これにより、クライアントアプリ（Next.js側）でSWRキャッシュ戦略を使用した際に、「裏側でのフェッチ通信」が絶対にCDNやブラウザの古いキャッシュを掴まないように強制しています。
