# フロントエンド詳細設計書 (Kondate Navi WebApp)

ユーザーが直接触れるWebアプリケーション・PWAの設計詳細です。Next.js 16 (App Router) を採用し、モバイルでの快活な操作性とSEO性能を両立させています。

---

## 1. 爆速表示を実現する独自キャッシュ戦略 (SWR)

- **場所**: `webapp/app/page.tsx` の `useEffect` 内
- **アーキテクチャ**: **「クライアント・サイド・フェッチ」**
  - Firebaseの関数などを介さず、ユーザーのブラウザが直接 Storage 上の静的ファイルをダウンロードします。
- **ロジック**:
  1. **LocalStorage優先読み込み**: ページロード時、まず `localStorage` に保存されている前回取得済みのJSONデータを即座に画面に表示します（表示速度 0ms）。
  2. **バックグラウンド・フェッチ**: 画面にデータが表示されている裏で、Firebase Storageへ最新のJSONを取りに行きます。
  3. **同期と更新**: 最新データが取得でき次第、メモリ上の `state` と `localStorage` の両方を上書き更新します。これにより、ユーザーは「待たされることなく」「常に最新の」献立を確認できます。

### 1.1 ストレージ容量への配慮
ブラウザの `localStorage` は5〜10MB程度の容量制限がありますが、本アプリの献立データは1ヶ月わずか `20KB` 程度です。1年分溜まっても `0.24MB` 程度（写真1枚分以下）のため、ユーザーのスマホ容量を圧迫するリスクはありません。

## 2. 外部データの取得とURL構造

### 2.1 取得先URLの動的組み立て
アプリは環境変数および表示中の年月を元に、フェッチ先のURLを動的に生成します。

- **ベースURL**: 環境変数 `NEXT_PUBLIC_MENU_BASE_URL` を使用します（デフォルトはローカルの `/data`）。
- **ファイル名**: ターゲット年月の `YYYY_M` 形式（例: `2026_3`）に `.json` を付与します。
- **Firebase Storage対応**: 
  ベースURLに `firebasestorage.googleapis.com` が含まれる場合、通常のREST APIアクセスのため、末尾に `?alt=media` パラメータを自動付与してブラウザがJSONとして正しくダウンロードできるように制御しています。

### 2.2 キャッシュ無効化の強制
Fetch APIのオプションとして `{ cache: 'no-store' }` を指定しています。これにより、サーバー（Firebase）側で月ごとにファイルが差し替えられた際、ブラウザキャッシュに邪魔されることなく確実に最新データを取得できます。

## 3. 画面表示のロジック

### 3.1 学校と調理場のマッピング
- **場所**: `webapp/app/page.tsx` の `SCHOOL_MAPPING` 定数
- **詳細**: 浦安市内の全小学校（17校）と、それに対応する給食センター（第一・第二調理場）を紐づけています。

### 3.2 メニューの特定ロジック（高速検索）
- **キーの構造**: 
  データは「年月（1ファイル）」の中に「全日程 × 全調理場」が詰まった配列形式になっています。特定の日のメニューを特定するには、以下の3つの情報を組み合わせています。
  1. **ファイル名 (年月)**: `2026_3.json` など。これで全データから1ヶ月分に絞り込まれます（月によるシャーディング）。
  2. **調理場名 (facility_name)**: 「第一調理場」など。
  3. **日付 (date)**: 「24日」など。
- **検索アルゴリズム**: 
  配列内のデータ数は最大でも31日×2箇所＝約60件程度と極めて少量です。そのため、ブラウザ上での単純な `find` ループによる検索であっても計算コストは無視できるほど小さく、実質的に O(1) に近い体感速度で特定の献立を表示できます。


### 3.2 文脈を汲み取る絵文字マッピング法
- **場所**: `webapp/app/page.tsx` の `PRIORITIZED_MAPPING`
- **詳細**: AIが抽出した献立テキスト（例：「揚げパン」「とん汁」）に対して、適切な絵文字を自動付与する仕組みです。
- **優先順位制御**: 単純な部分一致ではなく、リストの「前方にあるキーワード」を優先的にマッチさせます。これにより、例えば「いちごジャムパン」において、「いちご（🍎）」よりも「パン（🍞）」を優先して表示させるといった、食事のメイン要素を優先的に可視化する設計になっています。

## 4. 最新のSEO対策とPWA

- **SEO設定 (`layout.tsx`)**: 
  - `Metadata API` を活用し、「浦安市」「給食」「献立」などのキーワードを戦略的に配置。
  - `robots.ts` および `sitemap.ts` を動的生成し、Google検索結果でのインデックス効率を最大化しています。
- **PWA (Progressive Web App)**:
  - `manifest.json` と iOS/Android 用の各種アイコンを完備。
  - ブラウザの「ホーム画面に追加」を行うことで、実アプリのような使い勝手を提供します。
