# フロントエンド詳細設計書 (Kondate Navi WebApp)

ユーザーが直接触れるWebアプリケーション・PWAの設計詳細です。Next.js 16 (App Router) を採用し、モバイルでの快活な操作性とSEO性能を両立させています。

---

## 1. 爆速表示を実現する独自キャッシュ戦略 (SWR)

- **場所**: `webapp/app/page.tsx` の `useEffect` 内
- **アーキテクチャ**: **「クライアント・サイド・フェッチ」**
  - サーバー側の処理を介さず、ユーザーのブラウザが直接 GitHub Pages 上の静的ファイルをダウンロードします。
- **ロジック**:
  1. **LocalStorage優先読み込み**: ページロード時、まず `localStorage` に保存されている前回取得済みのJSONデータを即座に画面に表示します（表示速度 0ms）。
  2. **バックグラウンド・フェッチ**: 画面にデータが表示されている裏で、GitHub Pages へ最新のJSONを取りに行きます。
  3. **同期と更新**: 最新データが取得でき次第、メモリ上の `state` と `localStorage` の両方を上書き更新します。これにより、ユーザーは「待たされることなく」「常に最新の」献立を確認できます。

### 1.1 ストレージ容量への配慮
ブラウザの `localStorage` は5〜10MB程度の容量制限がありますが、本アプリの献立データは1ヶ月わずか `20KB` 程度です。1年分溜まっても `0.24MB` 程度（写真1枚分以下）のため、ユーザーのスマホ容量を圧迫するリスクはありません。

## 2. 外部データの取得とURL構造

### 2.1 取得先URL（GitHub Pages）
アプリは環境変数および表示中の年月を元に、GitHub Pages 上の静的 JSON ファイルをフェッチします。

- **URL**: `https://droprop.github.io/kondate-navi/data/YYYY_M.json`
- **ベースURL**: 環境変数 `NEXT_PUBLIC_MENU_BASE_URL` が `https://droprop.github.io/kondate-navi/data/` に設定されています。
- **キャッシュ無効化**:
  Fetch APIのオプションとして `{ cache: 'no-store' }` を指定しています。これにより、GitHub 上でファイルが更新された際、ブラウザのキャッシュに邪魔されることなく確実に最新データを取得できます。

## 3. 画面表示のロジック

### 3.1 学校と調理場のマッピング
- **場所**: `webapp/app/page.tsx` の `SCHOOL_CATEGORIES` 定数
- **詳細**: 浦安市内の全小学校（17校）と中学校（10校相当・第三調理場）をカテゴリ別に管理しています。中学校は調理が共通であるため、設定画面では「浦安市内中学校すべて」として1つの選択肢に統合し、ユーザーの選択コストを最小化しています。

### 3.2 メニューの特定ロジック（高速検索）
- **キーの構造**: 
  データは「年月（1ファイル）」の中に「全日程 × 全調理場」が詰まった配列形式になっています。特定の日のメニューを特定するには、以下の3つの情報を組み合わせています。
  1. **ファイル名 (年月)**: `2026_3.json` など。これで全データから1ヶ月分に絞り込まれます（月によるシャーディング）。
  2. **調理場名 (facility_name)**: 「第一調理場」など。
  3. **日付 (date)**: 「24日」など。
- **検索アルゴリズム**: 
  配列内のデータ数は最大でも 31日 × 3調理場 ＝ 約93件程度と極めて少量です。そのため、ブラウザ上での単純な `find` ループによる検索であっても計算コストは無視できるほど小さく、実質的に O(1) に近い体感速度で特定の献立を表示できます。


### 3.3 視認性を極めた Expert-UI
- **主菜の強調**: 献立リストにおいて主菜（★）を最上位に配置し、左側にオレンジのアクセントバーを表示。
- **栄養成分表示の最適化**: 1行のコンパクトな表示。数値を `font-medium` に調整し、モバイルでの高い視認性と洗練されたデザインを両立。
- **牛乳の末尾ソート**: 献立リストの最後尾に必ず「牛乳」が表示されるようロジックを統一。
- **ボトムシート（学校設定）**: モバイルアプリらしい操作感を実現。画面占有率を 80% に制限し、アドレスバー等による見切れを防止。

## 4. 最新のSEO対策とPWA

- **SEO設定 (`layout.tsx`)**: 
  - `Metadata API` を活用し、「浦安市」「給食」「献立」などのキーワードを戦略的に配置。
  - `robots.ts` および `sitemap.ts` を動的生成し、Google検索結果でのインデックス効率を最大化しています。
- **PWA (Progressive Web App)**:
  - `manifest.json` と iOS/Android 用の各種アイコンを完備。
  - ブラウザの「ホーム画面に追加」を行うことで、実アプリのような使い勝手を提供します。
